# Flutter Riverpod Starter

![Flutter](https://img.shields.io/badge/Flutter-3.7+-02569B?style=flat&logo=flutter)
![Riverpod](https://img.shields.io/badge/Riverpod-3.0+-0175C2?style=flat)
![Architecture](https://img.shields.io/badge/Architecture-App%20%2B%20Core-success)
![License](https://img.shields.io/badge/License-MIT-purple)

A Flutter starter project with Riverpod-based composition, reusable `core` infrastructure, modular `features`, and a practical app shell in `app/`.

---

## What This Starter Includes

- Riverpod provider composition
- App-level theme, router, localization, accessibility, analytics, notifications, and biometrics wiring
- Reusable `core` services, contracts, adapters, and generic UI helpers
- Modular feature folders
- Localization support
- Feature flags
- Bootstrap scripts for renaming, feature generation, and localization workflows

---

## Quick Start

### Prerequisites

- Flutter SDK
- Dart SDK

### Install dependencies

```bash
flutter pub get
```

### Optional environment setup

```bash
make setup-env
```

### Generate code

```bash
dart run build_runner build --delete-conflicting-outputs
```

### Run the app

```bash
flutter run
```

### Verify project health

```bash
flutter analyze
flutter test
```

## Project Structure

```text
lib/
├── app/                # App shell, composition, and policy wiring
├── core/               # Reusable infrastructure, services, contracts, and generic UI
├── examples/           # Capability demos and starter examples
├── features/           # Feature modules
├── l10n/               # Localization delegates and helpers
└── main.dart           # App entry point
```

### Layering Guide

- **`core/`**
  - contracts
  - adapters
  - low-level services
  - generic UI helpers

- **`app/`**
  - shell configuration
  - provider composition
  - app-level defaults and wiring

- **`features/`**
  - feature UI
  - feature state
  - feature-specific flows

## Bootstrap Scripts

### Rename app

```bash
./rename_app.sh --app-name "My App" --package-name com.example.myapp
```

### Generate a feature scaffold

```bash
./generate_feature.sh --name user_profile
```

### Generate localization files

```bash
./generate_language.sh generate
```

### Add a new language

```bash
./generate_language.sh add id
```

## Notes

- This repository is optimized for internal starter reuse first.
- Some sample features and demos may still need pruning depending on your target product.
- The safest boundary is:
  - `core` for reusable infrastructure
  - `app` for wiring and policy
  - `features` for product-specific flows
